# FAKDU Manual End-to-End Checklist

คู่มือนี้ใช้สำหรับ “เช็คใช้งานจริงทุก flow แบบกดหน้า UI” หลังการแก้ไขระบบโหมดเครื่องพนักงาน + แลกแต้ม

---

## 0) เตรียมสภาพแวดล้อมทดสอบ

1. เปิดเซิร์ฟเวอร์ (เครื่องหลัก)
   ```bash
   python3 app.py
   ```
2. เช็คความพร้อมแบบอัตโนมัติก่อนเริ่มทดสอบจริง
   ```bash
   python3 scripts/check_python_runtime_ready.py
   ```
3. เช็คความพร้อม LAN Sync เพิ่ม (ลิงก์พนักงาน + local-db + realtime events)
   ```bash
   python3 scripts/check_lan_sync_ready.py
   ```
4. เตรียม 2 อุปกรณ์/2 browser profile
   - **เครื่องหลัก (Master)**: ใช้สำหรับตั้งค่าร้าน + อนุมัติคำขอ
   - **เครื่องลูก (Client/Staff)**: ใช้ขอเชื่อมด้วย PIN/QR และทดสอบโหมด
5. เปิดทั้งสองฝั่งที่ URL เดียวกัน เช่น:
   - `http://127.0.0.1:8000/index.html`

> แนะนำเปิดโหมด private/incognito สำหรับเครื่องลูก เพื่อลดผลจาก localStorage เก่า

---

## 1) Seed ข้อมูลพื้นฐานก่อนเริ่ม

### 1.1 เมนู
- เข้าหน้า **หลังร้าน > จัดการร้าน**
- เพิ่มเมนูอย่างน้อย 2 รายการ:
  - `ชาเย็น` ราคา 50
  - `กาแฟเย็น` ราคา 60

### 1.2 สมาชิก + แต้ม
- เข้าหน้า **ระบบ > จัดการสมาชิก**
- เพิ่มสมาชิก 1 คน:
  - ชื่อ: `ทดสอบสมาชิก`
  - เบอร์: `0811111111`
  - แต้มเริ่มต้น: `200`

### 1.3 ตั้งค่าแลกแต้ม
- กลับไป **หลังร้าน > จัดการร้าน > เมนูที่ใช้สิทธิ์แลกแต้มได้**
- ติ๊กเปิดสิทธิ์ให้ `ชาเย็น` และตั้ง `100` แต้ม/ชิ้น
- กด “บันทึกแต้ม”

**Expected**
- เห็นสถานะเมนูว่า “เปิดสิทธิ์แล้ว”
- ตัวนับ “ใช้สิทธิ์ได้ / ยังไม่เปิดสิทธิ์” อัปเดตถูกต้อง

---

## 2) Flow เชื่อมเครื่องลูกด้วย PIN

### 2.1 เครื่องหลัก
- ไปหน้า **ระบบ**
- ดู PIN เชื่อมต่อ (6 หลัก) จากส่วน sync/เชื่อมอุปกรณ์

### 2.2 เครื่องลูก
- ไปหน้า **ระบบ > เชื่อมเครื่องพนักงาน**
- เลือกโหมด `โหมดลูกค้า (รับออเดอร์)`
- กรอก PIN 6 หลักจากเครื่องหลัก
- กด “เชื่อมต่อ”

### 2.3 เครื่องหลักอนุมัติ
- เมื่อมีคำขอเด้งเข้า inbox/approval
- กดอนุมัติแบบ `โหมดลูกค้า`

**Expected**
- เครื่องลูกเห็นเฉพาะแท็บตามโหมดลูกค้า (เข้า flow ลูกค้าได้)
- ไม่เกิด error เรื่อง session/token

---

## 3) Flow เชื่อมเครื่องลูกด้วย QR

### 3.1 เครื่องลูก
- หน้า **ระบบ > เชื่อมเครื่องพนักงาน**
- กดปุ่ม “สแกน QR”
- สแกน QR ของเครื่องหลัก

### 3.2 อนุมัติจากเครื่องหลัก
- ในรายการคำขอ กดอนุมัติแบบ `โหมดเช็คบิล`

**Expected**
- เครื่องลูกเข้าได้ตามโหมดเช็คบิล
- สถานะคำขอจาก pending -> approved

---

## 4) Flow อนุมัติโหมดผิดแล้วแก้ใหม่

1. ให้เครื่องลูกส่งคำขอใหม่
2. เครื่องหลักตั้งใจอนุมัติผิดโหมด (เช่น เช็คบิล)
3. ตรวจพฤติกรรมแท็บเครื่องลูกว่าโดนจำกัดตามโหมดที่ได้รับจริง
4. logout/reconnect แล้วอนุมัติใหม่เป็นอีกโหมด

**Expected**
- การจำกัดแท็บเปลี่ยนตามโหมดล่าสุดที่อนุมัติ
- ไม่ค้างโหมดเก่าหลัง reconnect

---

## 5) Flow รับออเดอร์ + เช็คบิลพื้นฐาน

### 5.1 รับออเดอร์
- เลือกโต๊ะ
- เพิ่ม `ชาเย็น` ลงตะกร้า 1 รายการ
- ส่งออเดอร์

### 5.2 เช็คบิล
- เข้า “คิวเช็คบิล”
- เปิดหน้าชำระเงิน
- ปิดบิลด้วยเงินสด

**Expected**
- รายการขายเพิ่มในประวัติ
- โต๊ะกลับสถานะว่างหลังปิดบิล

---

## 6) Flow แลกแต้ม

1. เปิด checkout ของโต๊ะที่มีรายการ
2. ในส่วนสมาชิก กรอกเบอร์ `0811111111`
3. กดปุ่ม “🎁 แลกแต้ม”
4. เลือก `ชาเย็น` ที่ตั้งค่าแลกไว้ (เช่น 1 ชิ้น)
5. ยืนยันเพิ่มรายการแลกแต้มเข้าบิล
6. ปิดบิล

**Expected**
- เพิ่มรายการ `(แลกแต้ม)` ในบิล
- ราคาแถวแลกแต้มเป็น 0 และแต้มถูกใช้ตามที่ตั้งค่า
- แต้มสมาชิกลดลงถูกต้องหลังปิดบิล

---

## 7) กรณีผิดพลาดที่ต้องลอง (Negative Cases)

1. ใส่ PIN ไม่ครบ 6 หลักแล้วกดเชื่อมต่อ
2. แลกแต้มโดยไม่กรอกสมาชิก
3. แลกแต้มเกินแต้มคงเหลือ
4. ติ๊กเปิดสิทธิ์แลกแต้มแต่ไม่ใส่แต้ม (>0)

**Expected**
- ทุกกรณีต้องมีข้อความแจ้งเตือนที่เข้าใจง่าย
- ไม่ทำให้ข้อมูลเสีย/ค้าง
